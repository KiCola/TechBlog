## 1. 环境准备

### 1.1 安装必要工具

在开始之前，确保你的电脑安装了以下工具：

#### Node.js（必须）

1. 访问 https://nodejs.org
2. 下载 **LTS 版本**（推荐 18.x 或 20.x）
3. 安装完成后，打开命令提示符验证：

```cmd
node --version
npm --version
```

看到版本号即为安装成功。

#### Git（必须）

1. 访问 https://git-scm.com/download/win
2. 下载并安装（全程默认选项即可）
3. 安装完成后验证：

```cmd
git --version
```

#### 安装博客依赖

打开命令提示符（`Win + R` 输入 `cmd`），执行：

```cmd
cd /d E:\科研\技术博客
npm install
```

等待安装完成（约 1-2 分钟）。

## 3. 写作与发布文章

### 3.1 创建新文章

所有文章存放在项目的 `posts/` 目录下，每篇文章是一个 Markdown 文件（`.md`）。

**步骤：**

1. 在 `E:\科研\技术博客\posts\` 目录下新建一个 `.md` 文件
2. 文件名即为文章的 URL，建议使用英文和连字符，例如：`my-first-post.md`
3. 按照以下格式编写文章

### 3.2 文章格式

每篇文章必须以 **frontmatter**（元数据头部）开始：

```markdown
---
title: 我的第一篇文章
date: 2026-03-17
category: 前端开发
tags: [React, JavaScript, 教程]
---

## 文章正文从这里开始

这里写你的技术内容...
```

**frontmatter 字段说明：**

| 字段 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| `title` | ✅ 必填 | 文章标题 | `我学会了 React Hooks` |
| `date` | ✅ 必填 | 发布日期（YYYY-MM-DD） | `2026-03-17` |
| `category` | 可选 | 文章分类（只能一个） | `前端开发` |
| `tags` | 可选 | 标签列表（可多个） | `[React, Hooks, JavaScript]` |

### 3.3 Markdown 语法速查

```markdown
# 一级标题
## 二级标题
### 三级标题

**加粗文字**
*斜体文字*
~~删除线~~

- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2

> 这是一段引用文字

[链接文字](https://example.com)

![图片描述](图片URL或路径)

`行内代码`

​```javascript
// 代码块（支持语法高亮）
function hello() {
  console.log('Hello World');
}
​```

| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
```

### 3.4 在文章中插入图片

**方式一：使用网络图片链接**

```markdown
![图片描述](https://example.com/image.png)
```

**方式二：使用本地图片**

1. 将图片放入 `E:\科研\技术博客\public\images\` 目录
2. 在文章中引用：

```markdown
![图片描述](/images/your-image.png)
```

**方式三：使用图床（推荐）**

推荐使用以下免费图床，上传图片后获取链接：
- [SM.MS](https://sm.ms) - 免费图床
- [Imgur](https://imgur.com) - 国际图床
- GitHub 仓库 Issues 上传图片（免费且稳定）

## 4. 部署到 GitHub Pages

### 4.1 注册 GitHub 账号

如果还没有账号，访问 https://github.com 注册。

### 4.2 创建 GitHub 仓库

有两种仓库命名方式，效果不同：

**方式 A：个人主页（推荐）**

- 仓库名格式：`你的用户名.github.io`
- 例如：用户名是 `zhangsan`，则仓库名为 `zhangsan.github.io`
- 访问地址：`https://zhangsan.github.io`

**方式 B：项目子路径**

- 仓库名可以随意，例如：`my-blog`
- 访问地址：`https://zhangsan.github.io/my-blog`
- 需要额外配置 `basePath`（见下方说明）

**创建仓库步骤：**

1. 登录 GitHub，点击右上角 `+` → `New repository`
2. 填写仓库名（推荐用方式 A）
3. 选择 **Public**（公开，GitHub Pages 免费托管需要）
4. **不要**勾选 Initialize this repository
5. 点击 `Create repository`

### 4.3 配置项目

**如果使用方式 A（个人主页），无需修改配置。**

**如果使用方式 B（项目子路径），需修改 `next.config.js`：**

打开 `E:\科研\技术博客\next.config.js`，修改为：

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    unoptimized: true,
  },
  basePath: '/my-blog',        // 替换为你的仓库名
  assetPrefix: '/my-blog/',    // 替换为你的仓库名
  trailingSlash: true,
};

module.exports = nextConfig;
```

### 4.4 初始化 Git 并推送代码

打开命令提示符，执行以下命令（替换 `你的用户名` 和 `仓库名`）：

```cmd
cd /d E:\科研\技术博客

:: 初始化 Git 仓库
git init

:: 配置 Git 用户信息（首次使用需要）
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"

:: 添加所有文件
git add .

:: 创建第一次提交
git commit -m "初始化博客"

:: 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

:: 推送到 GitHub
git branch -M main
git push -u origin main
```

**推送时会弹出 GitHub 登录窗口**，使用你的 GitHub 账号登录即可。

### 4.5 启用 GitHub Pages

1. 打开你的 GitHub 仓库页面
2. 点击顶部 **Settings**（设置）
3. 左侧菜单找到 **Pages**
4. 在 **Source** 下选择：
   - Branch: `gh-pages`
   - Folder: `/ (root)`
5. 点击 **Save**

> **注意**：第一次需要等待 GitHub Actions 完成构建（约 2-5 分钟）才会出现 `gh-pages` 分支。

### 4.6 等待自动部署

推送代码后，GitHub Actions 会自动：

1. 安装依赖（`npm install`）
2. 构建静态文件（`npm run build`）
3. 将 `out/` 目录推送到 `gh-pages` 分支
4. GitHub Pages 自动发布

**查看部署进度：**

1. 进入 GitHub 仓库
2. 点击 **Actions** 标签
3. 查看最新的 workflow 运行状态
4. 出现绿色 ✅ 即部署成功

**访问你的博客：**
- 方式 A：`https://你的用户名.github.io`
- 方式 B：`https://你的用户名.github.io/仓库名`

## 5. 自动部署配置（推荐）

项目已内置 GitHub Actions 配置文件（`.github/workflows/deploy.yml`），每次你 push 代码到 `main` 分支，博客会**自动构建并部署**，无需任何手动操作。

### 确认 Actions 权限

1. 进入 GitHub 仓库 → **Settings** → **Actions** → **General**
2. 找到 **Workflow permissions**
3. 选择 **Read and write permissions**
4. 点击 **Save**

这一步是让 GitHub Actions 有权限写入 `gh-pages` 分支。

## 6. 日常更新流程

部署完成后，以后每次更新博客只需要三步：

### 步骤一：写文章

在 `posts/` 目录下新建或编辑 `.md` 文件。

### 步骤二：本地预览（可选）

```cmd
cd /d E:\科研\技术博客
npm run dev
```

打开 http://localhost:3000 确认效果。

### 步骤三：提交并推送

```cmd
cd /d E:\科研\技术博客

:: 添加所有改动
git add .

:: 提交（描述这次改动内容）
git commit -m "新增文章：xxx"

:: 推送到 GitHub（自动触发部署）
git push
```

**2-5 分钟后**，访问你的博客即可看到新文章。

### 日常命令速查

```cmd
:: 查看改动状态
git status

:: 添加所有改动
git add .

:: 提交
git commit -m "提交说明"

:: 推送
git push

:: 拉取最新代码（多设备同步时使用）
git pull
```

## 7. 博客个性化配置

### 7.1 修改博客标题和描述

编辑 `app/layout.js`：

```javascript
export const metadata = {
  title: {
    default: '你的博客名称',        // 修改这里
    template: '%s | 你的博客名称',
  },
  description: '你的博客描述',      // 修改这里
};
```

### 7.2 修改导航栏标题

编辑 `components/Navbar.js`，找到以下部分修改：

```jsx
<span className="brand-name">TechBlog</span>  {/* 改为你的名字 */}
```

同时修改 GitHub 链接为你的 GitHub 主页：

```jsx
<a href="https://github.com/你的用户名" ...>
  ⭐ GitHub
</a>
```

### 7.3 修改主题颜色

编辑 `styles/globals.css`，修改 `:root` 中的颜色变量：

```css
:root {
  --bg-primary: #0d1117;    /* 主背景色 */
  --bg-card: #1c2128;       /* 卡片背景色 */
  --accent: #58a6ff;        /* 强调色（链接、按钮等） */
  --text-primary: #e6edf3;  /* 主文字颜色 */
  --text-secondary: #8b949e; /* 次要文字颜色 */
}
```

### 7.4 添加自定义域名（可选）

如果你有自己的域名（如 `www.yourname.com`）：

1. 在 `public/` 目录下创建 `CNAME` 文件，内容为你的域名：

```
www.yourname.com
```

2. 在你的域名 DNS 控制台添加以下记录：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| CNAME | www | 你的用户名.github.io |

3. 在 GitHub 仓库 Settings → Pages → Custom domain 填入你的域名
4. 勾选 **Enforce HTTPS**

## 8. 常见问题排查

### Q1：npm install 安装失败

**解决方案：**

```cmd
:: 清除缓存后重试
npm cache clean --force
npm install
```

如果网络不好，可以切换到国内镜像：

```cmd
npm config set registry https://registry.npmmirror.com
npm install
```

### Q2：npm run dev 报错

常见原因：
- Node.js 版本过低 → 升级到 18.x 以上
- 依赖未安装 → 先运行 `npm install`
- 端口 3000 被占用 → 关闭其他程序或用 `npm run dev -- -p 3001`

### Q3：git push 失败

**原因 1：未配置用户信息**

```cmd
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱"
```

**原因 2：未登录 GitHub**

```cmd
:: 使用个人访问令牌（PAT）
:: 前往 GitHub → Settings → Developer Settings → Personal access tokens
:: 生成 token，推送时密码填写 token 内容
```

**原因 3：远程仓库地址错误**

```cmd
:: 查看当前远程地址
git remote -v

:: 修改远程地址
git remote set-url origin https://github.com/你的用户名/仓库名.git
```

### Q4：GitHub Actions 部署失败

1. 进入仓库 → Actions → 点击失败的 workflow 查看日志
2. 常见问题：
   - **权限不足**：Settings → Actions → General → Workflow permissions 改为 Read and write
   - **构建错误**：查看日志中的具体错误信息，通常是代码语法问题

### Q5：部署成功但页面样式丢失

通常是 `basePath` 配置问题。如果仓库名不是 `用户名.github.io`，需要在 `next.config.js` 配置 `basePath`：

```javascript
const nextConfig = {
  output: 'export',
  images: { unoptimized: true },
  basePath: '/仓库名',
  assetPrefix: '/仓库名/',
  trailingSlash: true,
};
```

### Q6：文章不显示或显示旧内容

- 本地开发：刷新浏览器即可
- GitHub Pages：等待 2-5 分钟让 CDN 缓存刷新，或强制刷新（`Ctrl + Shift + R`）
