<h2>1. 快速开始</h2>
<h3>环境要求</h3>
<ul>
<li>Node.js 18.x 或以上</li>
<li>Git</li>
</ul>
<h3>安装与运行</h3>
<pre><code class="language-bash"># 克隆仓库
git clone https://github.com/KiCola/TechBlog.git
cd TechBlog

# 安装依赖
npm install

# 启动本地开发服务器
npm run dev
</code></pre>
<p>打开浏览器访问 <a href="http://localhost:3000">http://localhost:3000</a></p>

<h2>2. 写博客文章</h2>
<p>所有文章存放在 <code>posts/</code> 目录下，每篇文章是一个 <code>.md</code> 文件。</p>
<h3>创建新文章</h3>
<p>在 <code>posts/</code> 目录下新建一个 Markdown 文件，文件名即为文章的 URL：</p>
<pre><code>posts/
├── my-first-post.md        → /posts/my-first-post
├── react-hooks-guide.md    → /posts/react-hooks-guide
└── algorithm-dp.md         → /posts/algorithm-dp
</code></pre>
<p><strong>文件名规范：</strong></p>
<ul>
<li>使用英文和连字符（<code>-</code>）</li>
<li>全部小写</li>
<li>不要有空格或中文</li>
<li>例如：<code>deep-learning-intro.md</code>、<code>cpp-pointer-notes.md</code></li>
</ul>
<h3>文章模板</h3>
<pre><code class="language-markdown">---
title: 文章标题
date: 2026-03-17
category: 分类名称
tags: [标签1, 标签2, 标签3]
cover: https://图片URL（可选）
---

## 第一个章节

正文内容从这里开始...
</code></pre>
<hr>
<h2>3. 文章元数据说明</h2>
<p>每篇文章开头的 <code>---</code> 包裹部分叫做 <strong>frontmatter</strong>，用于设置文章属性：</p>
<table>
<thead>
<tr>
<th>字段</th>
<th>是否必填</th>

<th>说明</th>

<td><code>title</code></td>

<td>✅ 必填</td>

</tr>
<tr>

<td>可选</td>

<h3>分类（category）</h3>
<p>分类用于首页的筛选按钮，同一篇文章只能属于一个分类：</p>
<pre><code class="language-yaml">category: 前端开发
</code></pre>
<p>常用分类参考：<code>前端开发</code>、<code>后端开发</code>、<code>算法</code>、<code>机器学习</code>、<code>工具使用</code>、<code>读书笔记</code>、<code>随笔</code></p>
<h3>标签（tags）</h3>
<p>标签显示在文章卡片和详情页底部，一篇文章可以有多个标签：</p>
<pre><code class="language-yaml"># 单个标签
tags: [JavaScript]

# 多个标签
tags: [React, TypeScript, 前端工程化]

# 也可以这样写
tags:
  - React
  - TypeScript
  - Hooks
</code></pre>

<h2>6. 封面图设置</h2>
<p>在文章 frontmatter 中添加 <code>cover</code> 字段：</p>
<pre><code class="language-yaml">---
title: 我的文章
date: 2026-03-17
cover: https://images.unsplash.com/photo-xxx?w=800&amp;q=80
---
</code></pre>
<h3>使用网络图片（推荐）</h3>
<p>从以下免费图库获取图片链接：</p>
<ul>
<li><a href="https://unsplash.com">Unsplash</a> - 高质量免费图片，右键图片 → 复制图片地址</li>
<li><a href="https://pexels.com">Pexels</a> - 免费商用图片</li>
</ul>
<p>Unsplash 图片链接可以加参数控制尺寸：</p>
<pre><code>https://images.unsplash.com/photo-xxx?w=800&amp;q=80
                                       ↑宽度  ↑质量(0-100)
</code></pre>
<h3>使用本地图片</h3>
<ol>
<li>将图片放入 <code>public/images/</code> 目录</li>
<li>在 frontmatter 中引用：</li>
</ol>
<pre><code class="language-yaml">cover: /images/my-cover.jpg
</code></pre>
<p><strong>注意</strong>：部署到 GitHub Pages 后，本地图片路径需要加 <code>/TechBlog</code> 前缀：</p>
<pre><code class="language-yaml">cover: /TechBlog/images/my-cover.jpg
</code></pre>
<h3>不设置封面</h3>
<p>不填写 <code>cover</code> 字段时，系统会根据文章分类自动生成渐变色占位封面。</p>

<h2>7. 更换背景图</h2>
<p>背景图在 <code>app/layout.js</code> 中配置，本地和线上路径自动区分。</p>
<h3>使用本地图片</h3>
<ol>
<li>将背景图片放入 <code>public/</code> 目录，例如 <code>public/bg.jpg</code></li>
<li>打开 <code>app/layout.js</code>，找到第 6-10 行，修改文件名：</li>
</ol>
<pre><code class="language-javascript">const bgStyle = `
  body {
    background-image:
      linear-gradient(rgba(13, 17, 23, 0.82), rgba(13, 17, 23, 0.82)),
      url(&#39;${basePath}/bg.jpg&#39;);   // ← 修改这里的文件名
  }
`;
</code></pre>
<h3>使用网络图片</h3>
<p>直接填写完整 URL，不需要 <code>basePath</code>：</p>
<pre><code class="language-javascript">const bgStyle = `
  body {
    background-image:
      linear-gradient(rgba(13, 17, 23, 0.82), rgba(13, 17, 23, 0.82)),
      url(&#39;https://images.unsplash.com/photo-xxx?w=1920&amp;q=80&#39;);
  }
`;
</code></pre>
<h3>调整遮罩透明度</h3>
<p><code>rgba(13, 17, 23, 0.82)</code> 中最后的数字控制遮罩深度：</p>
<ul>
<li><code>0.6</code> → 图片较亮，文字可能不清晰</li>
<li><code>0.82</code> → 默认，图片较暗，文字清晰</li>
<li><code>0.92</code> → 图片很暗，接近纯色背景</li>
</ul>

<h2>8. 新建自定义页面</h2>
<h3>新建普通页面</h3>
<p>在 <code>app/</code> 目录下新建文件夹和 <code>page.js</code>，路径即为 URL：</p>
<pre><code>app/about/page.js   →   /about
app/projects/page.js →  /projects
</code></pre>
<p><strong>示例：新建「关于我」页面</strong></p>
<p>创建 <code>app/about/page.js</code>：</p>
<pre><code class="language-jsx">import &#39;../styles/post.css&#39;;

export const metadata = {
  title: &#39;关于我&#39;,
};

export default function AboutPage() {
  return (
    &lt;div className=&quot;post-detail&quot;&gt;
      &lt;article className=&quot;post-article&quot;&gt;
        &lt;header className=&quot;post-header&quot;&gt;
          &lt;h1 className=&quot;post-title&quot;&gt;关于我&lt;/h1&gt;
        &lt;/header&gt;
        &lt;div className=&quot;post-content&quot;&gt;
          &lt;p&gt;这里写你的自我介绍...&lt;/p&gt;
          &lt;p&gt;研究方向：xxx&lt;/p&gt;
          &lt;p&gt;联系方式：your@email.com&lt;/p&gt;
        &lt;/div&gt;
      &lt;/article&gt;

  );

</code></pre>

  &lt;Link href=&quot;/about&quot;&gt;关于我&lt;/Link&gt;        {/* 新增 */}

<h3>修改导航栏 Logo</h3>
<p>编辑 <code>components/Navbar.js</code>：</p>
<pre><code class="language-jsx">&lt;span className=&quot;brand-name&quot;&gt;TechBlog&lt;/span&gt;  {/* ← 改为你的名字 */}
</code></pre>
<h3>修改 GitHub 链接</h3>
<p>编辑 <code>components/Navbar.js</code>：</p>
<pre><code class="language-jsx">&lt;a href=&quot;https://github.com/你的用户名&quot; ...&gt;
</code></pre>
<h3>修改主题颜色</h3>
<p>编辑 <code>styles/globals.css</code> 中的 CSS 变量：</p>
<pre><code class="language-css">:root {
  --accent: #58a6ff;        /* 主强调色（链接、按钮、标签）*/

  --accent-warm: #f78166;   /* 警告/删除色 */
  --bg-primary: #0d1117;    /* 主背景色 */
  --bg-card: #1c2128;       /* 卡片背景色 */
  --text-primary: #e6edf3;  /* 主文字色 */
  --text-secondary: #8b949e;/* 次要文字色 */
}
</code></pre>
<h3>修改页脚文字</h3>
<p>编辑 <code>app/layout.js</code>：</p>
<pre><code class="language-jsx">&lt;p&gt;© {new Date().getFullYear()} 你的名字 · 你的口号&lt;/p&gt;
</code></pre>

<h2>10. 部署到 GitHub Pages</h2>
<h3>自动部署（每次 push 自动触发）</h3>
<p>项目已配置 GitHub Actions，每次推送到 <code>main</code> 分支自动部署。</p>
<p><strong>首次配置：</strong></p>
<ol>
<li>进入 GitHub 仓库 → <strong>Settings</strong> → <strong>Actions</strong> → <strong>General</strong></li>
<li><strong>Workflow permissions</strong> 选择 <code>Read and write permissions</code> → <strong>Save</strong></li>
<li>进入 <strong>Settings</strong> → <strong>Pages</strong></li>
<li>Source 设为 <code>gh-pages</code> 分支，<code>/ (root)</code> → <strong>Save</strong></li>
</ol>
<p><strong>之后每次更新只需：</strong></p>
<pre><code class="language-bash">git add .
git commit -m &quot;更新说明&quot;
git push
</code></pre>
<p>2-5 分钟后访问 <a href="https://inzeroworld.com.cn/TechBlog/">https://inzeroworld.com.cn/TechBlog/</a> 查看更新。</p>

<pre><code>① 在 posts/ 目录新建或编辑 .md 文件
        ↓
② npm run dev  →  http://localhost:3000 预览（可选）
        ↓
③ git add .
   git commit -m &quot;新增文章：xxx&quot;
   git push
        ↓
④ GitHub Actions 自动构建（2-5 分钟）
        ↓
⑤ 访问 https://inzeroworld.com.cn/TechBlog/ 查看 ✅
</code></pre>

git add .               # 添加所有改动

git commit -m &quot;说明&quot;    # 提交
git push                # 推送
git log --oneline       # 查看提交历史
</code></pre>

<h2>常见问题</h2>
<p><strong>Q: 文章不显示？</strong>
A: 检查文件名是否为英文、frontmatter 格式是否正确（特别是 <code>date</code> 字段格式）。</p>
<p><strong>Q: LaTeX 公式显示不正常？</strong>
A: 确保公式内的 <code>_</code> 和 <code>^</code> 没有被 Markdown 解析，可以用 <code>\</code> 转义特殊字符。</p>
<p><strong>Q: 本地正常但线上样式乱了？</strong>
A: 检查 <code>next.config.js</code> 中的 <code>basePath</code> 是否为 <code>/TechBlog</code>。</p>
<p><strong>Q: 推送后多久能看到更新？</strong>
A: 通常 2-5 分钟，可在仓库 Actions 标签查看进度。</p>
<p><strong>Q: 如何删除文章？</strong>
A: 删除 <code>posts/</code> 目录下对应的 <code>.md</code> 文件，然后 push。</p>
